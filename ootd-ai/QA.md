# QA 체크리스트

> 배포 전 또는 주요 기능 변경 시 아래 항목을 순서대로 확인하세요.
> 모바일(375px 기준)과 데스크탑(1280px 이상) 각각 확인 필요.

---

## 1. 인증 (Auth)

- [ ] 로그인 페이지 진입 시 소셜 로그인 버튼 정상 표시
- [ ] 로그인 후 홈(/)으로 리다이렉트
- [ ] 브라우저 닫고 재접속 시 로그인 상태 유지 (세션 지속)
- [ ] 비로그인 상태에서 `/wardrobe`, `/settings` 접근 시 로그인 페이지로 이동
- [ ] 로그아웃 후 랜딩 페이지로 이동

---

## 2. 온보딩

- [ ] 신규 가입 후 `/onboarding` 자동 진입
- [ ] 닉네임 입력 후 저장
- [ ] 프로필 이미지 업로드 및 저장
- [ ] 건너뛰기 선택 후 홈으로 이동
- [ ] 온보딩 완료 후 다시 `/onboarding` 접근 시 홈으로 리다이렉트

---

## 3. OOTD 분석 — 모바일

- [ ] 카메라 버튼으로 사진 촬영 후 분석 시작
- [ ] 갤러리 버튼으로 사진 선택 후 분석 시작
- [ ] 사진 선택 즉시 스캐닝 애니메이션 표시
- [ ] fetch() 응답 즉시 (Gemini 첫 토큰 전) 결과 패널 열림 + 스켈레톤 표시
- [ ] 스트리밍 중 점수·헤드라인 순서대로 채워짐
- [ ] 총점 배지 사진 우측 상단 오버레이 표시
- [ ] 헤드라인 사진 하단 텍스트 표시
- [ ] 하단 시트: 카테고리 4개 배지 (핏, 컬러, 스타일링, 날씨) 짤리지 않고 표시
- [ ] 잘된 점 / 개선점 / 팁 텍스트 정상 표시
- [ ] 날씨 적합성 한 줄 표시
- [ ] 저장 버튼 클릭 시 "저장 중" 표시 후 완료 토스트 (스캐닝 화면으로 돌아가지 않음)
- [ ] 다시 분석 버튼 동작
- [ ] 일일 분석 한도 초과 시 429 에러 토스트 표시

## 4. OOTD 분석 — 데스크탑

- [ ] 파일 드래그 앤 드롭으로 사진 등록 후 분석 시작
- [ ] 클릭 업로드로 사진 등록 후 분석 시작
- [ ] 결과 패널: 총점 원형 + 헤드라인 정상 표시
- [ ] 카테고리 배지 4개 (핏·실루엣 / 컬러 조합 / 스타일링 / 날씨 적합) 나란히 표시
- [ ] 잘된 점 / 개선점 좌우 나란히 표시
- [ ] Stylist Tips 섹션 정상 표시
- [ ] 날씨 적합성 섹션 정상 표시
- [ ] 스켈레톤 → 실제 데이터 전환 자연스러움

## 5. 옷장 자동 등록 (Add-Clothes)

- [ ] OOTD 분석 결과에서 "옷장에 자동 등록하기" 버튼 표시 (분석 완료 후, 스트리밍 종료 후)
- [ ] 버튼 클릭 시 `/add-clothes`로 이동 + 이미지 자동 로드
- [ ] 이미지 로드 완료 후 AI 스캔 자동 시작
- [ ] 스캔 진행: 1/3 위치 분석 → 2/3 배경 제거 → 3/3 이름 생성 순서 표시
- [ ] 추출된 아이템 카드 표시 (이미지 + 이름 + 카테고리)
- [ ] 이름 탭해서 수정 가능
- [ ] 카테고리 드롭다운으로 변경 가능
- [ ] X 버튼으로 개별 아이템 제거
- [ ] "옷장에 저장하기" 버튼 클릭 후 `/wardrobe`로 이동
- [ ] 배경 제거 실패 아이템은 크롭 이미지로 fallback 표시
