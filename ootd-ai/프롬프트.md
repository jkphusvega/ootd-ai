### Supabase 초기화 파일 제작

> Next.js App Router 환경에서 Supabase 클라이언트를 서버/클라이언트 각각 초기화하는 파일을 만들어줘.
>
> 1. lib/supabase/server.ts — 서버 컴포넌트용 (createServerClient, cookies)
> 2. lib/supabase/client.ts — 클라이언트 컴포넌트용 (createBrowserClient)
> 3. lib/supabase/middleware.ts — updateSession 함수로 세션 자동 갱신
>
> @supabase/ssr 패키지를 사용하고, 환경변수는 NEXT_PUBLIC_SUPABASE_URL / NEXT_PUBLIC_SUPABASE_ANON_KEY를 사용.

---

### Supabase 세션 유지 설정 (proxy.ts)

> Next.js 16에서 middleware.ts 대신 proxy.ts를 사용해서 모든 요청마다 Supabase 세션을 자동 갱신해줘.
> lib/supabase/middleware.ts의 updateSession을 호출하고, matcher는 정적 파일과 이미지를 제외한 모든 경로로 설정.

### Supabase DB 테이블 설계

> 아래 테이블을 Supabase SQL Editor에서 생성하는 SQL을 작성해줘.
>
> 1. user_profiles — user_id(FK), nickname, profile_image, height, weight, fit_preference, style_moods(text[]), body_goal, api_usage(jsonb), share_id, is_public
> 2. clothes — id, user_id(FK), category(outer/tops/bottoms/shoes/bag/accessory/socks/ootd_feed), name, image_url, created_at
> 3. push_subscriptions — id, user_id(FK), endpoint, p256dh, auth, created_at (웹 푸시 구독 정보)
>
> 모든 테이블에 RLS를 활성화하고, user_id = auth.uid() 조건으로 본인 데이터만 접근 가능하도록 정책 추가.
> 일일 사용량은 별도 테이블 없이 user_profiles.api_usage JSONB에 `{action_YYYY-MM-DD: count}` 형태로 저장.

### OOTD 분석 API 구현

> app/api/analyze-ootd/route.ts를 만들어줘.
>
> 역할: 사용자가 업로드한 OOTD 사진을 Gemini 멀티모달 API로 분석하고 결과를 스트리밍으로 반환.
>
> 요구사항:
> - POST 요청, body: { imageBase64, weatherInfo: { temperature, condition }, userProfile }
> - Gemini systemInstruction: 서울 기반 패션 에디터 페르소나, 친근하고 솔직한 어조, 평균 65점 기준 엄격 채점
> - 응답 JSON 스키마: score, fit, color, styling, weather(0-100), headline(20자 이내), strengths(2개), improvements(2개), tips(2개), weatherNote
> - generateContentStream()으로 스트리밍 응답
> - 모델: gemini-3.5-flash 우선, 503/UNAVAILABLE 시 gemini-2.5-flash 자동 전환 (각 모델 최대 2회 재시도)
> - thinkingConfig는 gemini-2.5-flash 전용 (다른 모델에 보내면 Invalid Argument 에러 발생하므로 모델명 분기 처리)
> - Supabase로 유저 인증 확인 및 일일 사용량 제한 적용 (analyze-ootd: 20회/일)

### 스트리밍 분석 훅 구현

> hooks/useOotdAnalysis.ts를 만들어줘.
>
> 역할: OOTD 분석 전체 상태와 로직을 관리하는 커스텀 훅.
>
> 상태:
> - scanState: 'idle' | 'scanning' | 'success' | 'error'
> - partialCritique: 스트리밍 중 부분적으로 도착한 분석 결과
> - critique: 최종 완성된 분석 결과
> - isStreaming, isRateLimited, isSaving
>
> 로직:
> 1. 이미지 선택 시 Canvas API로 768px WebP 압축
> 2. /api/analyze-ootd 호출
> 3. fetch() 응답 즉시(Gemini 첫 토큰 전) 패널 오픈 — setScanState('success'), setPartialCritique({})
> 4. ReadableStream 청크를 누적하며 정규식으로 부분 JSON 파싱 → setPartialCritique 업데이트
> 5. 스트림 완료 후 JSON.parse로 최종 결과 setCritique
> 6. handleSaveToFeed: 분석 결과를 Supabase Storage + clothes 테이블(category='ootd_feed')에 저장
>    isSaving 상태 분리로 저장 중 스캐닝 화면이 재노출되지 않도록 처리

### AI 코디 큐레이션 API 구현

> app/api/curate-outfit/route.ts를 만들어줘.
>
> 역할: 사용자 옷장 데이터, 날씨, 프로필, 착용 이력을 기반으로 Gemini가 오늘의 코디를 추천.
>
> 입력: { weatherInfo, userProfile, occasion }
> - occasion: 'daily'(일상) / 'work'(출근) / 'date'(데이트) / 'outdoor'(야외활동) / 'formal'(격식)
>
> 처리 순서:
> 1. Supabase에서 해당 유저의 옷장 아이템 전체 조회 (ootd_feed 제외)
> 2. lib/behaviorContext.ts의 getUserBehaviorContext()로 착용 이력·스타일 패턴 조회
> 3. 옷장 목록 + 날씨 + occasion + 프로필(핏 선호, 바디 목표, 스타일 임베딩) + 행동 컨텍스트를 Gemini에 전달
>
> 출력 JSON: { title, description, style, colorTone, items: [{ category, name, image_url, reason }] }
> - items의 name/image_url은 실제 옷장 아이템에서 선택
> - 클라이언트에서 name 기준으로 무신사·29CM 검색 URL 생성
>
> 모델: gemini-3.5-flash 우선, 과부하 시 gemini-2.5-flash 자동 전환
> 일일 사용량 제한: curate-outfit 15회/일

>   - suggestions: 추천 아이템 3-5개, 각각 이름/카테고리/추천 이유/가격대/브랜드 팁
