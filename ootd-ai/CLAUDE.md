# 프로젝트 최종 목표

## 현재 단계 (진행 중)
돈을 쓰기 전에 기능과 플로우를 제대로 완성하는 것.
- 모든 주요 기능 구현 (분석, 큐레이션, 옷장, 스타일리스트 채팅)
- 에러 핸들링, 빈 상태 UI, 온보딩 경험 완성
- 첫 사용자가 가치를 바로 느낄 수 있는 UX 완성

## 출시 이후 단계 (최종 목표)
돈을 투자해서 핵심 부분을 고도화:
- 분석 속도 개선 (더 빠른 모델 또는 캐싱 전략)
- 분석 품질 고도화 (더 정교한 프롬프트 또는 파인튜닝)
- 인프라 안정성 확보

## 해자(Moat) 전략
경쟁 우위는 모델이 아니라 **축적된 사용자 데이터**:
- like/dislike/worn 피드백 → 다음 추천에 실제로 반영하는 피드백 루프
- 개인별 착장 히스토리 (6개월+ 쌓이면 이탈 불가)
- 사용자가 많아질수록 추천 품질이 올라가는 네트워크 효과
- 크롤링/직접 학습은 비용 대비 효과 낮음 — 사용자 피드백 데이터가 핵심

## 기술 스택
- Next.js 16 App Router
- Supabase (auth + DB)
- Gemini API (분석 + 스타일리스트 채팅)
- Vercel 배포

## Gemini 모델 목록 (실존 모델)
`gemini-3.5-flash`, `gemini-2.5-flash`, `gemini-2.5-pro`, `gemini-2.0-flash` 모두 실존하는 모델임.
`gemini-3.5-flash`가 존재하지 않는다고 절대 단정하지 말 것.
